## Tech Debt

### Phase 1: Foundation

- `paths.ts` uses `console.warn/error` instead of Logger utility (inconsistent logging; messages go to dev console not OutputChannel)
- `jsonlParser.ts` uses inline `require(path)` and `require(os)` instead of top-level imports

### Phase 5: Webview Dashboard

- `UsageChart.tsx` uses `any` type for Recharts callback parameters (library typing limitation)
- No React error boundaries around tab components (unhandled render errors could crash entire dashboard)

### General

- Config change listener refreshes on ANY `claude-usage.*` setting change (could debounce or filter to specific settings)
- Burn rate tracker is module-level state in extension.ts (could be encapsulated in SessionWatcher)
- webview.js bundle is 2.0 MB (Recharts is large; consider lazy loading or lighter chart library in v2)

**Total: 7 items across 3 categories. None are blockers.**
