<task type="auto">
  <name>Task 2: Configure dual esbuild bundling and JSX support</name>
  <files>esbuild.config.mjs, tsconfig.json</files>
  <action>
Update `esbuild.config.mjs` to build BOTH extension and webview bundles:

1. Rename current config to `extensionConfig` (keep all existing settings, change `outdir` to `outfile: 'dist/extension.js'` for explicit single-file output).

2. Add a `webviewConfig`:
```javascript
const webviewConfig = {
  entryPoints: ['src/webview/app/index.tsx'],
  bundle: true,
  outfile: 'dist/webview.js',
  format: 'iife',
  platform: 'browser',
  target: 'es2020',
  loader: { '.tsx': 'tsx', '.ts': 'ts', '.css': 'css' },
  sourcemap: !production,
  minify: production,
  logLevel: 'info',
};
```

3. Update the main() function to build both configs. For watch mode, create two contexts and watch both. For build mode, use `Promise.all` to build both in parallel.

4. Update `tsconfig.json` to enable JSX:
- Add `"jsx": "react-jsx"` to compilerOptions (this uses the automatic React JSX transform, no need for `import React` in every file)
- Add `"jsxImportSource": "react"` if needed for the automatic transform

Create a placeholder `src/webview/app/index.tsx` file with just a comment (`// Webview entry point - implemented in Plan 05-03`) so the build doesn't fail on missing entry point.
  </action>
  <verify>
Run `npm run build` and confirm both `dist/extension.js` and `dist/webview.js` are generated without errors.
Run `npm run compile` (tsc --noEmit) to confirm TypeScript is happy with JSX config.
  </verify>
  <done>

  </done>
