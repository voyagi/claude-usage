**index.tsx:**
1. Declare `acquireVsCodeApi` function type
2. Call `acquireVsCodeApi()` once and export as `vscode` (singleton -- only one call allowed)
3. Import `createRoot` from `react-dom/client`
4. Import `App` and `app.css`
5. Mount with `createRoot(document.getElementById('root')!).render(<App />)`

**App.tsx:**
1. Define tab type: `type Tab = 'overview' | 'trends' | 'session';`
2. Use `useState<Tab>('overview')` for active tab (overview is default per CONTEXT.md decision)
3. Use `useState<DashboardData | null>(null)` for data
4. Use `useEffect` to:
   a. Send `{ type: 'requestData' }` message to extension on mount
   b. Add `window.addEventListener('message', handler)` that listens for `usageData` messages and updates state
   c. Return cleanup to remove the listener
5. Use VS Code `setState`/`getState` for tab persistence:
   - On tab change: `vscode.setState({ activeTab })`
   - On mount: check `vscode.getState()?.activeTab` to restore tab selection
6. Render structure:
   - `<TrustIndicator />` at top
   - `<nav className="tabs">` with three tab buttons (Overview, Trends, Session)
   - Active tab button gets `tab-active` class, others get `tab-inactive`
   - `<main>` with conditional rendering: overview -> `<OverviewTab data={data} />`, trends -> placeholder div "Trends tab (Plan 05-04)", session -> placeholder div "Session tab (Plan 05-05)"
   - If `data` is null, show a loading skeleton (simple pulsing placeholder divs)

**app.css:**
Create comprehensive CSS using VS Code CSS variables. Include:

1. **Base styles:** body background, font, color using `--vscode-sideBar-*` variables
2. **Tab navigation:** flex row, border-bottom, active/inactive colors using `--vscode-panelTitle-*`
3. **Tab content area:** padding, overflow auto
4. **Cards/sections:** bordered boxes using `--vscode-panel-border`
5. **Trust indicator:** small badge with lock icon
6. **Progress bar styles:** track, fill with safe/warning/critical color variants
7. **Summary numbers:** large text for key metrics
8. **Token breakdown:** grid/flex layout for the four token types
9. **Loading skeleton:** pulsing animation with theme-appropriate colors
10. **Utility classes:** truncate, muted text, spacing

Color tokens for token types (must work light + dark):
- Input tokens: `#4FC3F7` (light blue)
- Output tokens: `#81C784` (green)
- Cache creation: `#FFB74D` (orange)
- Cache read: `#CE93D8` (purple)

Define these as CSS custom properties in `:root` for reuse in both CSS and chart components.

Reusable component for rate limit proximity display. Props:
- `label: string` (e.g., "Session (5hr)")
- `current: number` (current token count)
- `limit: number` (estimated limit)
- `percentage: number` (0-100)
- `resetTime: string | null` (ISO string)
- `isHit: boolean`

Implementation:
1. Show label and "X / Y (Z%)" on one line (header)
2. Horizontal bar: track div with fill div. Fill width = percentage%.
3. Color classes: `progress-safe` (< 60%), `progress-warning` (60-95%), `progress-critical` (>= 95%)
4. If resetTime provided, show "Resets: Xh Ym" below the bar
5. Format tokens with commas (use Intl.NumberFormat)
6. If `isHit`, show "LIMIT HIT" badge

**OverviewTab.tsx:**

Layout (dense summary at top, details below -- per CONTEXT.md decision):
