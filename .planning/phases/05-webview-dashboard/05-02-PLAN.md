  <action>
Create `src/webview/app/types.ts` with all the data shapes the dashboard needs. This is the contract between extension and webview.

Define these types:

1. **DashboardData** -- the main data payload sent from extension to webview. Should contain all fields needed by all three tabs:
   - Token breakdown: inputTokens, outputTokens, cacheCreationTokens, cacheReadTokens (from current window/day/period)
   - Cost data: todayCost, monthCost, totalCost
   - Rate limits: session5h, weekly, weeklySonnet -- each with name, currentTokens, estimatedLimit, percentage, resetTime (as ISO string for serialization), isHit
   - Session timing: windowStart (ISO string | null), windowExpiry (ISO string | null), timeRemainingMinutes (number | null)
   - Burn rate: tokensPerMinute, minutesUntilLimit (number | null)
   - Time series data for charts: `trendData` -- array of `{ period: string; inputTokens: number; outputTokens: number; cacheCreationTokens: number; cacheReadTokens: number; totalCost: number; }` objects
   - Session comparison: currentSessionTokens, averageSessionTokens, sessionCount
   - Metadata: lastUpdated (ISO string), filesProcessed, linesSkipped, planType (string)

2. **RateLimitData** -- extracted sub-type for rate limit info (mirrors RateLimitInfo but with serializable dates):
   - name, currentTokens, estimatedLimit, percentage, resetTime (string | null), isHit

3. **TrendDataPoint** -- single data point for charts:
   - period, inputTokens, outputTokens, cacheCreationTokens, cacheReadTokens, totalCost

4. **WebviewMessage** -- messages FROM webview TO extension:
   - `{ type: 'requestData' }` -- initial data request
   - `{ type: 'changePeriod'; period: 'daily' | 'weekly' | 'monthly' }` -- period switch

5. **ExtensionMessage** -- messages FROM extension TO webview:
   - `{ type: 'usageData'; payload: DashboardData }` -- data update
   - `{ type: 'periodData'; payload: { period: string; data: TrendDataPoint[] } }` -- period-specific trend data

Use discriminated unions (type field) for message types. Export everything as named exports.

Create `src/webview/DashboardProvider.ts` implementing `vscode.WebviewViewProvider`.

**Class structure:**

```typescript
export class DashboardProvider implements vscode.WebviewViewProvider {
  public static readonly viewType = 'claude-usage.dashboardView';

  private _view?: vscode.WebviewView;
  private _currentData?: DashboardData;

  constructor(private readonly _extensionUri: vscode.Uri) {}

  resolveWebviewView(webviewView, context, token) { ... }

  // Public method for extension.ts to push data
  public updateData(data: DashboardData): void { ... }

  private _getHtmlForWebview(webview: vscode.Webview): string { ... }
  private _getNonce(): string { ... }
}
```

**resolveWebviewView implementation:**
1. Store reference to webviewView in `this._view`
2. Configure webview options: `enableScripts: true`, `localResourceRoots: [this._extensionUri]`
3. Set HTML content via `_getHtmlForWebview()`
4. Listen for messages from webview via `onDidReceiveMessage`:
   - `requestData`: If `_currentData` exists, post it back immediately
   - `changePeriod`: (Store the period preference, extension will handle the data refresh)
5. Handle visibility changes via `webviewView.onDidChangeVisibility`:
   - If webview becomes visible AND `_currentData` exists, post the data
6. Handle disposal: clear `this._view` reference

**_getHtmlForWebview:**
1. Generate nonce using `crypto.randomBytes(16).toString('base64')`  (import `crypto` from Node.js)
2. Get webview URIs for the bundled JS: `webview.asWebviewUri(vscode.Uri.joinPath(this._extensionUri, 'dist', 'webview.js'))`
3. Also resolve a CSS URI if we bundle CSS separately -- or inline the CSS reference in HTML
4. Return HTML template with:
   - Proper DOCTYPE and html lang
   - `<meta charset="UTF-8">`
   - CSP meta tag: `default-src 'none'; style-src ${webview.cspSource} 'unsafe-inline'; script-src 'nonce-${nonce}'; font-src ${webview.cspSource};`
   - `<div id="root"></div>`
   - `<script nonce="${nonce}" src="${scriptUri}"></script>`

**updateData method:**
1. Store data in `this._currentData`
2. If `this._view` exists and is visible, post message: `{ type: 'usageData', payload: data }`

Do NOT import from `../types.js` for vscode types -- use `import * as vscode from 'vscode'`. Import DashboardData and message types from `./app/types.js`.
