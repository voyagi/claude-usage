const DEFAULT_PRICING: Record<string, ModelPricing> = {

**calculateCost(usage: TokenUsage, pricing: Record&lt;string, ModelPricing&gt;): number**
- Look up model in pricing table
- If model not found: log warning, use claude-sonnet-4-5 as fallback (most common model in Claude Code)
- Calculate:
  - inputCost = (inputTokens / 1_000_000) * inputPerMillion
  - outputCost = (outputTokens / 1_000_000) * outputPerMillion
  - For cache creation cost: use cacheCreation5m and cacheCreation1h from TokenUsage
    - cache5mCost = (cacheCreation5m / 1_000_000) * inputPerMillion * cache5mWriteMultiplier
    - cache1hCost = (cacheCreation1h / 1_000_000) * inputPerMillion * cache1hWriteMultiplier
    - If neither 5m nor 1h breakdown available (both 0) but cacheCreationTokens > 0: default to 5m multiplier
  - cacheReadCost = (cacheReadTokens / 1_000_000) * inputPerMillion * cacheReadMultiplier
  - Return sum of all costs

**getDefaultPricing(): Record&lt;string, ModelPricing&gt;**
- Returns deep copy of DEFAULT_PRICING

**loadPricingFromConfig(): Record&lt;string, ModelPricing&gt;**
- Read from vscode.workspace.getConfiguration('claude-usage').get('pricing')
- If user has overrides: deep merge with defaults (user overrides win per-model)
- If no overrides: return defaults
- Validate overrides with Zod schema (ModelPricing shape) — log warning and use default if invalid

Important: All pricing MUST be configurable via VS Code settings so users can update rates without waiting for an extension update. This satisfies requirement TP-04.

  <action>
Create time bucket aggregation that groups TokenUsage records into session, daily, weekly, and monthly buckets.

**aggregateUsage(records: TokenUsage[]): TimeBuckets**
- Create empty TimeBuckets (4 Maps)
- For each record in records:
  - **Session bucket**: key = record.sessionId
  - **Daily bucket**: key = format(startOfDay(record.timestamp), 'yyyy-MM-dd')
    - Use local timezone (not UTC) for calendar boundaries — this matches how users think about "today"
  - **Weekly bucket**: key = format(startOfWeek(record.timestamp, { weekStartsOn: 1 }), "yyyy-'W'II")
    - weekStartsOn: 1 = Monday (ISO week standard)
  - **Monthly bucket**: key = format(startOfMonth(record.timestamp), 'yyyy-MM')
  - For each bucket: use createEmptyAggregatedUsage() and addToAggregation() from tokenCounter.ts

**mergeTimeBuckets(a: TimeBuckets, b: TimeBuckets): TimeBuckets**
- Merge two TimeBuckets together (additive)
- For matching keys: add values together
- For non-matching keys: include both
- Update firstMessage/lastMessage correctly (min/max)
- This is used when aggregating across multiple parse rounds

**getTimeBucketSummary(buckets: TimeBuckets): { totalSessions: number; totalDays: number; totalCost: number; totalMessages: number }**
- Quick summary stats from the buckets
- totalSessions = buckets.session.size
- totalDays = buckets.daily.size
- totalCost = sum of all daily bucket costs
- totalMessages = sum of all daily bucket messageCount

**serializeTimeBuckets(buckets: TimeBuckets): SerializedTimeBuckets**
- Convert Maps to [key, value][] arrays for JSON serialization (globalState)

**deserializeTimeBuckets(serialized: SerializedTimeBuckets): TimeBuckets**
- Convert [key, value][] arrays back to Maps
- Reconstruct Date objects from ISO strings in firstMessage/lastMessage

Import date-fns functions: startOfDay, startOfWeek, startOfMonth, format.
Import from tokenCounter: createEmptyAggregatedUsage, addToAggregation.
Import types from types.ts.
