  <name>Task 1: Create streaming JSONL parser with error recovery</name>
  <files>src/parser/jsonlParser.ts</files>
  <action>
Create a streaming JSONL parser using Node.js readline + createReadStream.

**parseSessionFile(filePath: string, logger: Logger): Promise&lt;FileParseResult&gt;**
- Open file with fs.createReadStream (encoding: 'utf8') — do NOT use exclusive locks
- Create readline interface with crlfDelay: Infinity (handles both \r\n and \n)
- For each line:
  - Skip empty lines (whitespace-only)
  - Try JSON.parse, catch errors (truncated lines expected during active sessions)
  - On parse error: increment linesSkipped counter, log warning with file path and line snippet (first 100 chars), continue
  - Check if top-level type === 'assistant' — skip all other types
  - Pass to parseAssistantMessage() from schemas.ts for Zod validation
  - If validation returns null (missing usage data): skip silently
  - If validation succeeds: call extractTokenUsage() to build TokenUsage record
- Return FileParseResult with filePath, records[], linesSkipped count, error messages

**parseAllSessions(logger: Logger): Promise&lt;{ records: TokenUsage[]; filesProcessed: number; linesSkipped: number; errors: string[] }&gt;**
- Call findAllSessionFiles() from paths.ts to discover all JSONL files
- Parse each file with parseSessionFile()
- Aggregate all records, track total files processed and lines skipped
- If a file is unreadable (EACCES, EBUSY, etc.): log warning, skip the file, add to errors, continue to next
- Sort all records by timestamp ascending before returning

Important anti-patterns to AVOID:
- Do NOT use fs.readFileSync or read entire file into memory
- Do NOT use exclusive file locks or flags that prevent concurrent writes
- Do NOT fail the entire parse if one line is corrupt
- Do NOT fail the entire parse if one file is unreadable

<task type="auto">
  <name>Task 2: Create token extraction and billable token calculation</name>
  <files>src/parser/tokenCounter.ts</files>
  <action>
Create token extraction logic that converts parsed JSONL messages into TokenUsage objects.

**extractTokenUsage(parsed: validated assistant message, sessionId: string): TokenUsage**
- Extract from message.usage:
  - inputTokens: usage.input_tokens (uncached input)
  - outputTokens: usage.output_tokens
  - cacheCreationTokens: usage.cache_creation_input_tokens (default 0)
  - cacheReadTokens: usage.cache_read_input_tokens (default 0)
  - cacheCreation5m: usage.cache_creation?.ephemeral_5m_input_tokens (default 0)
  - cacheCreation1h: usage.cache_creation?.ephemeral_1h_input_tokens (default 0)
- Parse timestamp from ISO 8601 string to Date
- Extract model from message.model
- Use sessionId from top-level JSONL field
- Set cost to 0 initially (calculated later by pricing module)
- Return TokenUsage object

**getBillableTokenCount(usage: TokenUsage): number**
- Returns inputTokens + cacheCreationTokens
- cache_read_input_tokens do NOT count toward rate limits (for Claude 4.x models)
- Output tokens are separate from input token rate limits
- Add a code comment explaining WHY cache reads are excluded (rate limiting is cache-aware)

**getTotalTokens(usage: TokenUsage): number**
- Returns inputTokens + outputTokens + cacheCreationTokens + cacheReadTokens
- This is the "total activity" number, not the billable number

**createEmptyAggregatedUsage(): AggregatedUsage**
- Returns a zero-initialized AggregatedUsage object
- All numeric fields 0, firstMessage and lastMessage null, messageCount 0

**addToAggregation(target: AggregatedUsage, source: TokenUsage): void**
- Adds source token counts and cost to target
- Increments messageCount
- Updates firstMessage (min) and lastMessage (max) timestamps

Export all functions as named exports.
