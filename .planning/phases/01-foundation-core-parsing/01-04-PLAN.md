**class UsageStore:**

Constructor: takes vscode.ExtensionContext

**async saveUsageData(buckets: TimeBuckets, stats: { filesProcessed: number; linesSkipped: number }): Promise&lt;void&gt;**
- Serialize TimeBuckets using serializeTimeBuckets()
- Build PersistedState object:
  - version: 1 (for future schema migrations)
  - lastParseTimestamp: new Date().toISOString()
  - totalFilesProcessed: stats.filesProcessed
  - totalLinesSkipped: stats.linesSkipped
  - timeBuckets: serialized buckets
- Call this.context.globalState.update('claudeUsage', state)

**async loadUsageData(): Promise&lt;{ buckets: TimeBuckets; stats: { filesProcessed: number; linesSkipped: number } } | null&gt;**
- Call this.context.globalState.get&lt;PersistedState&gt;('claudeUsage')
- If null: return null (first run)
- Check version field — if not 1, log warning and return null (force reparse on schema change)
- Deserialize timeBuckets using deserializeTimeBuckets()
- Return deserialized data

**async clearUsageData(): Promise&lt;void&gt;**
- Call this.context.globalState.update('claudeUsage', undefined)
- Log "Usage data cleared" to output channel

**getLastParseTimestamp(): string | null**
- Quick read of lastParseTimestamp without full deserialization
- Use globalState.get to read just the timestamp

Export UsageStore class.

**activate(context: vscode.ExtensionContext): void**

1. Create Logger instance ("Claude Usage Monitor")
2. Log "Claude Usage Monitor activating..."
3. Create UsageStore with context
4. Create status bar item (vscode.window.createStatusBarItem):
   - alignment: StatusBarAlignment.Right
   - priority: 100
   - text: "$(loading~spin) Claude Usage: Loading..."
   - show() immediately
   - Push to context.subscriptions

5. Register "claude-usage.clearData" command:
   - Call store.clearUsageData()
   - Reset status bar text to "Claude Usage: No data"
   - Show information message "Claude Usage: Data cleared. Reload to reparse."

6. Run initial parse (async, non-blocking — do NOT await in activate):
   ```
   performInitialParse(store, logger, statusBarItem).catch(err => {
     logger.error(`Initial parse failed: ${err.message}`);
     statusBarItem.text = "$(warning) Claude Usage: Error";
     statusBarItem.tooltip = `Parse failed: ${err.message}`;
   });
   ```

7. Push statusBarItem and command disposable to context.subscriptions

**async performInitialParse(store, logger, statusBarItem): Promise&lt;void&gt;**

1. Log "Starting full JSONL parse..."
2. Check if ~/.claude/projects/ exists — if not, set status bar to "Claude Usage: No data directory found", return
3. Call parseAllSessions(logger)
4. Load pricing with loadPricingFromConfig()
5. Apply cost to each record: record.cost = calculateCost(record, pricing)
6. Call aggregateUsage(records) to build TimeBuckets
7. Save to store: store.saveUsageData(buckets, { filesProcessed, linesSkipped })
8. Get summary: getTimeBucketSummary(buckets)
9. Update status bar:
   - text: "$(cloud) Claude: $X.XX | Xtok" (total cost and total tokens abbreviated)
   - tooltip: Multi-line string showing:
     - "Claude Usage Monitor"
     - "Today: $X.XX (X,XXX tokens)"
     - "This month: $X.XX (X,XXX tokens)"
     - "Sessions parsed: N"
     - "Files processed: N (N lines skipped)"
     - "Last updated: {timestamp}"
10. Log summary stats to output channel

**deactivate(): void**
- Log "Claude Usage Monitor deactivated"
- (subscriptions are automatically disposed by VS Code)

Important implementation notes:
- activate() MUST return quickly (VS Code requirement) — parsing is async
- Status bar shows loading spinner during parse, then switches to data
- If projects dir doesn't exist, degrade gracefully (not an error)
- Format cost as "$X.XX", format large token counts with K/M suffixes (e.g., "1.2M tok", "456K tok")
- Push ALL disposables to context.subscriptions
