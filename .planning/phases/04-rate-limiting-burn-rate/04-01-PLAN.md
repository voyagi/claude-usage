<feature>
  <name>Burn Rate Calculator with EMA Smoothing</name>
  <files>src/core/burnRate.ts, src/core/burnRate.test.ts</files>
  <behavior>
    The module provides:

    1. `BurnRateTracker` interface:
       - `ema: number` (current EMA value, tokens/min)
       - `lastUpdate: Date` (last calculation timestamp)
       - `alpha: number` (smoothing factor, default 0.2)

    2. `createBurnRateTracker(alpha?: number): BurnRateTracker`
       - Creates tracker with ema=0, lastUpdate=now, configurable alpha

    3. `calculateBurnRateEMA(buckets: TimeBuckets, tracker: BurnRateTracker, lookbackMinutes: number): { rate: number; tracker: BurnRateTracker }`
       - Sums output tokens from sessions with `lastMessage >= (now - lookbackMinutes)`
       - Uses actual time span (earliest active firstMessage to now) for rate calculation
       - Applies EMA formula: `newEma = alpha * currentRate + (1 - alpha) * oldEma`
       - Returns 0 if no activity in lookback window (idle state)
       - Returns updated tracker (immutable — creates new object)
       - Does NOT use `subHours` for minutes — use `subMinutes` from date-fns

    4. `predictTimeUntilLimit(currentTokens: number, limitTokens: number, burnRatePerMin: number): number | null`
       - Returns null if burnRate is 0 (can't predict when idle)
       - Returns 0 if currentTokens >= limitTokens (already hit)
       - Returns `(limitTokens - currentTokens) / burnRatePerMin` otherwise

    Test cases:
    - `calculateBurnRateEMA` with empty buckets -> rate: 0
    - `calculateBurnRateEMA` with activity in lookback window -> positive rate
    - `calculateBurnRateEMA` with activity OUTSIDE lookback window -> rate: 0
    - EMA smoothing: second call blends with previous value
    - `predictTimeUntilLimit(100, 1000, 10)` -> 90 (minutes)
    - `predictTimeUntilLimit(100, 1000, 0)` -> null
    - `predictTimeUntilLimit(1000, 1000, 10)` -> 0
    - `predictTimeUntilLimit(1100, 1000, 10)` -> 0
    - `createBurnRateTracker()` -> alpha 0.2, ema 0
    - `createBurnRateTracker(0.5)` -> alpha 0.5
  </behavior>
  <implementation>
    Pure TypeScript module importing only from date-fns and ../types.ts.
    No VS Code dependencies. Use `subMinutes`, `differenceInMinutes` from date-fns.
    Floor burn rate at 0 (never negative). Cap minutes remaining at a reasonable max (e.g., 999999) to avoid Infinity display.
  </implementation>
</feature>

<feature>
  <name>Tier Detection from Credentials</name>
  <files>src/core/tierDetection.ts, src/core/tierDetection.test.ts</files>
  <behavior>
    The module provides:

    1. `parseCredentialsFile(content: string): { rateLimitTier?: string; subscriptionType?: string } | null`
       - Parses JSON content, returns relevant fields
       - Returns null on JSON parse error (graceful degradation)

    2. `detectTierFromCredentials(credentials: { rateLimitTier?: string; subscriptionType?: string } | null, fallback: PlanType): PlanType`
       - null credentials -> returns fallback
       - rateLimitTier includes 'max_20' -> 'max20'
       - rateLimitTier includes 'max_5' -> 'max5'
       - subscriptionType === 'pro' -> 'pro'
       - No match -> returns fallback

    Test cases:
    - `parseCredentialsFile('{"rateLimitTier":"default_claude_max_5x"}')` -> { rateLimitTier: "default_claude_max_5x" }
    - `parseCredentialsFile('invalid json')` -> null
    - `parseCredentialsFile('{}')` -> {}
    - `detectTierFromCredentials({ rateLimitTier: "default_claude_max_5x" }, 'pro')` -> 'max5'
    - `detectTierFromCredentials({ rateLimitTier: "default_claude_max_20x" }, 'pro')` -> 'max20'
    - `detectTierFromCredentials({ subscriptionType: "pro" }, 'max5')` -> 'pro'
    - `detectTierFromCredentials(null, 'max5')` -> 'max5'
    - `detectTierFromCredentials({}, 'max5')` -> 'max5'
    - `detectTierFromCredentials({ rateLimitTier: "unknown_tier" }, 'max5')` -> 'max5'
  </behavior>
  <implementation>
    Pure TypeScript module importing only PlanType from ../types.ts.
    No VS Code dependencies, no fs calls (file reading happens at the call site).
    Use case-insensitive matching for rateLimitTier string checks.
  </implementation>
</feature>
