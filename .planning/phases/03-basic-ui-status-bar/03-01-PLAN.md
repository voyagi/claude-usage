<task type="auto">
  <name>Task 2: Create rate limit calculation engine</name>
  <files>src/core/rateLimits.ts</files>
  <action>
  Create `src/core/rateLimits.ts` with a `calculateRateLimits` function.

  The function signature:
  ```typescript
  export function calculateRateLimits(
    buckets: TimeBuckets,
    planType: PlanType
  ): RateLimitStatus
  ```

  Implementation:
  1. Get plan config via `getPlanConfig(planType)` to read token limits.
  2. **Session 5hr limit:** Sum output tokens from all session buckets where `lastMessage` is within the last 5 hours of now. This approximates the rolling 5-hour window. If `sessionTokenLimit` is not set on the plan, return percentage 0 and `isHit: false`.
  3. **Weekly limit:** Sum output tokens from the current ISO week bucket (key format `yyyy-'W'II`). Compare against `weeklyTokenLimit`.
  4. **Weekly Sonnet limit:** For now, use the same weekly bucket data (we don't have per-model aggregation in TimeBuckets yet — Phase 4 concern). Compare against `weeklySonnetLimit`. Add a TODO comment noting this should filter by model name once per-model weekly aggregation exists.
  5. **Reset times:**
     - Session 5hr: Calculate as `oldestSessionMessage + 5 hours` (the earliest message in the 5hr window determines when tokens start expiring). If no messages in window, resetTime = null.
     - Weekly: Calculate as `startOfWeek(now, {weekStartsOn: 1}) + 7 days` using date-fns.
     - Weekly Sonnet: Same as weekly reset.
  6. **Percentage:** `Math.min(100, (currentTokens / estimatedLimit) * 100)`. Round to nearest integer.
  7. **worstPercentage:** `Math.max(session5h.percentage, weekly.percentage, weeklySonnet.percentage)`.

  Also export a helper:
  ```typescript
  export function calculateBurnRate(buckets: TimeBuckets): number
  ```
  - Look at session buckets with `lastMessage` within the last 10 minutes.
  - Sum their output tokens.
  - Calculate tokens per minute = totalRecentTokens / minutesElapsed.
  - Return 0 if no recent activity (nothing in last 10 minutes).

  And a builder function:
  ```typescript
  export function buildStatusBarData(
    buckets: TimeBuckets,
    stats: { filesProcessed: number; linesSkipped: number },
    planType: PlanType
  ): StatusBarData
  ```
  - Aggregates total input/output tokens from all daily buckets
  - Gets today's cost and this month's cost from buckets
  - Calls calculateRateLimits and calculateBurnRate
  - Returns a complete StatusBarData object

  Import `startOfWeek, addDays, differenceInMinutes, subHours` from date-fns. Import types from `../types.js`. Import `getPlanConfig` from `../pricing/plans.js`.
  </action>

  Functions to implement:

  1. `formatTokens(tokens: number): string` — Smart abbreviation:
     - < 1,000: show exact number ("500")
     - >= 1,000 and < 10,000: one decimal ("1.2K")
     - >= 10,000 and < 1,000,000: no decimal ("45K")
     - >= 1,000,000: one decimal ("3.4M")
     Note: Slightly enhanced from existing version — adds 1 decimal for 1K-9.9K range.

  2. `formatTokensExact(tokens: number): string` — Full number with commas for tooltip:
     - Uses Intl.NumberFormat('en-US') for comma-separated display
     - E.g. "1,234,567"

  3. `formatCooldown(resetTime: Date | null): string` — Duration until reset:
     - null: return "" (no active limit)
     - Past: return "Ready"
     - < 1 hour: return "Xm" (e.g. "34m")
     - >= 1 hour: return "Xh Ym" (e.g. "2h 34m")

  4. `formatCost(cost: number): string` — Smart cost display:
     - < $0.01: return "$0.00"
     - < $1.00: two decimals ("$0.42")
     - < $100: two decimals ("$12.50")
     - >= $100: no decimals ("$150")

  5. `formatPercentage(percent: number): string` — Usage percentage:
     - Round to nearest integer, append "%" (e.g. "73%")

  6. `formatBurnRate(tokensPerMin: number): string` — Tokens per minute:
     - 0: return "" (no display when inactive)
     - < 100: exact number + "/min" (e.g. "42/min")
     - >= 100: abbreviated + "/min" (e.g. "1.2K/min")

  All functions must be pure (no side effects, no VS Code imports). Export all as named exports.
