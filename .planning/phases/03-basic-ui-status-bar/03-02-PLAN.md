<task type="auto">
  <name>Task 1: Create StatusBarManager class</name>
  <files>src/ui/statusBar.ts</files>
  <action>
  Create `src/ui/statusBar.ts` with a `StatusBarManager` class that manages two VS Code StatusBarItems.

  **Constructor:** `constructor(context: vscode.ExtensionContext)`
  - Create `metricsItem` via `vscode.window.createStatusBarItem('claude-usage.metrics', vscode.StatusBarAlignment.Right, 100)`
  - Create `cooldownItem` via `vscode.window.createStatusBarItem('claude-usage.cooldown', vscode.StatusBarAlignment.Right, 99)`
  - Both items: `command = 'claude-usage.showMenu'`
  - Both items: add to `context.subscriptions` for auto-disposal
  - Show both items immediately with loading state: metricsItem.text = `$(loading~spin) Claude: Loading...`
  - Read `claude-usage.compactMode` setting from workspace config, store as `private isCompactMode: boolean`

  **Method: `update(data: StatusBarData): void`**
  - This is the main render method, called whenever data changes.

  Metrics item text (normal mode):
  - Format: `$(cloud) ${formatCost(data.totalCost)} | ${formatPercentage(data.rateLimits.worstPercentage)}`
  - If burn rate > 0, append: ` | ${formatBurnRate(data.burnRate)}`
  - Example: `$(cloud) $4.20 | 73% | 1.2K/min`

  Metrics item text (compact mode):
  - Format: `$(cloud) ${formatCost(data.totalCost)} ${formatPercentage(data.rateLimits.worstPercentage)}`
  - Omit burn rate in compact mode
  - Example: `$(cloud) $4.20 73%`

  Cooldown item text:
  - Find the rate limit with `isHit === true` that has the soonest `resetTime`. If none are hit, find the limit with the highest percentage.
  - If any limit isHit: `$(clock) ${formatCooldown(soonestResetTime)}`
  - If no limit isHit but worstPercentage >= 60: `$(clock) ${formatCooldown(soonestResetOfWorstLimit)}`
  - If worstPercentage < 60: hide cooldown item (`cooldownItem.hide()`)
  - Example: `$(clock) 2h 34m`

  Background color (applied to BOTH items):
  - Use `data.rateLimits.worstPercentage` to determine color
  - >= 80: `new vscode.ThemeColor('statusBarItem.errorBackground')` (red)
  - >= 60: `new vscode.ThemeColor('statusBarItem.warningBackground')` (yellow)
  - < 60: `undefined` (default/green)

  Tooltip for metrics item (use vscode.MarkdownString for formatting):
  - Line 1: "**Claude Usage Monitor**"
  - Line 2: blank
  - Line 3: "**Today:** ${formatCost(data.todayCost)} | **Month:** ${formatCost(data.monthCost)}"
  - Line 4: blank
  - Line 5: "**Tokens:** ${formatTokensExact(data.totalInputTokens)} in / ${formatTokensExact(data.totalOutputTokens)} out"
  - Line 6: blank
  - Line 7: "**Rate Limits** _(estimated)_"
  - For each limit (session5h, weekly, weeklySonnet):
    - `"- ${limit.name}: ${formatPercentage(limit.percentage)} (${formatTokensExact(limit.currentTokens)} / ${formatTokensExact(limit.estimatedLimit)})"`
    - If limit.isHit: append " -- LIMIT HIT, resets ${formatCooldown(limit.resetTime)}"
    - If limit.resetTime and not hit: append " -- resets ${formatCooldown(limit.resetTime)}"
  - Line: blank
  - Burn rate line: if > 0, "**Burn Rate:** ${formatBurnRate(data.burnRate)}"
  - Last line: "Files: ${data.filesProcessed} | Updated: ${data.lastUpdated.toLocaleTimeString()}"
  - Set `tooltip.isTrusted = true` and `tooltip.supportHtml = false` (markdown only)

  Tooltip for cooldown item:
  - Brief: "Soonest rate limit reset. Click for actions."

  **Method: `showRefreshing(): void`**
  - Set metricsItem.text to `$(sync~spin) Refreshing...`
  - Clear backgroundColor to undefined
  - Store previous text/color to restore on error

  **Method: `showError(message: string): void`**
  - Set metricsItem.text to `$(warning) Claude: Error`
  - Set metricsItem.tooltip to error message
  - Set backgroundColor to errorBackground
  - Auto-restore after 5 seconds using setTimeout (store timer ref, clear on next update)

  **Method: `showNoData(): void`**
  - Set metricsItem.text to `$(cloud) Claude: No data`
  - Hide cooldown item

  **Method: `dispose(): void`**
  - Dispose both items (though context.subscriptions handles this, explicit is defensive)

  Export `StatusBarManager` as a named export.

  IMPORTANT: Do NOT import or call showUsageMenu directly from statusBar.ts. The command binding happens through VS Code's command registry — statusBar.ts just sets `.command = 'claude-usage.showMenu'` and extension.ts registers the command handler.
  </action>
  <verify>Run `npx tsc --noEmit` (via compile.ps1) — no type errors. Confirm StatusBarManager is exported from statusBar.ts.</verify>
  <done>StatusBarManager class exists with two StatusBarItems (metrics priority 100, cooldown priority 99), update() renders cost/percentage/burn rate/cooldown, color-coding uses ThemeColor for green/yellow/red, tooltip shows full breakdown with all three rate limits and exact numbers, compact mode abbreviates display, showRefreshing/showError/showNoData handle state transitions.</done>
