<task type="auto">
  <name>Task 1: Create SessionWatcher with debounced incremental parsing</name>
  <files>src/watcher/sessionWatcher.ts</files>
  <action>
Create `src/watcher/sessionWatcher.ts` with a `SessionWatcher` class that:

1. Constructor takes:
   - `context: vscode.ExtensionContext` (for subscriptions + globalState access)
   - `onUpdate: (buckets: TimeBuckets, stats: { filesProcessed: number; linesSkipped: number }) => void` — callback when data updates (extension.ts will use this to update status bar + persist)

2. Private members:
   - `watcher: vscode.FileSystemWatcher` — the file system watcher
   - `offsetTracker: OffsetTracker` — from Plan 01
   - `logger: Logger` — for diagnostics
   - `recentlyCreated: Set<string>` — tracks files created in last 1 second (debounce create+change race)
   - `debounceTimers: Map<string, NodeJS.Timeout>` — per-file debounce timers
   - `currentBuckets: TimeBuckets` — in-memory aggregated data (starts empty, gets merged on each update)
   - `totalFilesProcessed: number` — running count
   - `totalLinesSkipped: number` — running count
   - `pricing: Record<string, ModelPricing>` — loaded once in constructor

3. `start()` method:
   a. Load pricing via `loadPricingFromConfig()`
   b. Create watcher: `vscode.workspace.createFileSystemWatcher(pattern, false, false, true)`
      - Pattern: `new vscode.RelativePattern(getClaudeProjectsDir(), '**/*.jsonl')`
      - Watch creates and changes, ignore deletes
   c. Register `onDidCreate` handler:
      - Add filePath to `recentlyCreated` set
      - Remove from set after 1000ms via setTimeout
      - Call `handleFileChange(uri.fsPath, 0)` (parse from beginning)
   d. Register `onDidChange` handler:
      - If filePath is in `recentlyCreated`, skip (already handled by create)
      - Otherwise debounce: clear existing timer for this path, set new 500ms timer calling `handleFileChange(uri.fsPath)`
   e. Push watcher to `context.subscriptions` for automatic disposal
   f. Push each event listener disposable to `context.subscriptions`
   g. Log "SessionWatcher started, watching {projectsDir}"

4. Private `handleFileChange(filePath: string, forceOffset?: number)` method:
   a. Get offset: if `forceOffset !== undefined` use it, else `offsetTracker.getOffset(filePath)`
   b. Call `parseIncremental(filePath, offset, logger)`
   c. If result.records.length === 0 and result.linesSkipped === 0, return early (no new data)
   d. Apply pricing: for each record, `record.cost = calculateCost(record, pricing)`
   e. Aggregate new records: `const newBuckets = aggregateUsage(result.records)`
   f. Merge into current: `this.currentBuckets = mergeTimeBuckets(this.currentBuckets, newBuckets)`
   g. Update offset: `await offsetTracker.setOffset(filePath, result.newOffset)`
   h. Update stats: `totalFilesProcessed++` (or track unique files via a Set), `totalLinesSkipped += result.linesSkipped`
   i. Call `onUpdate(this.currentBuckets, { filesProcessed: totalFilesProcessed, linesSkipped: totalLinesSkipped })`
   j. Log: "Parsed {records.length} new records from {path.basename(filePath)}"

5. `setInitialBuckets(buckets: TimeBuckets, stats: { filesProcessed: number; linesSkipped: number })` method:
   - Called by extension.ts after initial full parse completes
   - Sets `this.currentBuckets = buckets` so incremental updates merge correctly
   - Sets `this.totalFilesProcessed = stats.filesProcessed`
   - Sets `this.totalLinesSkipped = stats.linesSkipped`

6. `dispose()` method:
   - Clear all debounce timers
   - Watcher is auto-disposed via context.subscriptions, but clear references
   - Log "SessionWatcher disposed"

7. Error handling:
   - Wrap `handleFileChange` body in try/catch
   - Log errors but never throw (watcher errors must not crash extension)
   - If file doesn't exist (deleted between event and handler), log and skip

Use named export. Strict TypeScript. No `any` types.
